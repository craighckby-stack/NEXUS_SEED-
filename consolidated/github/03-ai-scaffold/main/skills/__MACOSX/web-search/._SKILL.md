# Frontend Design Skill

A comprehensive skill for transforming UI style requirements into production-ready frontend code with systematic design tokens, accessibility compliance, and creative execution.

## Overview

This skill provides a structured approach to frontend development, ensuring consistency, maintainability, and scalability. It covers the entire process, from design token generation to component implementation, and includes tools for accessibility, testing, and debugging.

## Methodology

The frontend design skill is based on the following principles:

1. **Design Token System**: A semantic and organized system for managing visual properties, including colors, typography, spacing, and more.
2. **Component-Driven Development**: Building reusable components that can be assembled to create complex UI elements.
3. **Accessibility Compliance**: Ensuring that the frontend code meets accessibility standards and guidelines.
4. **Testing and Debugging**: Implementing testing strategies and debugging techniques to identify and resolve issues.

## Tools and Dependencies

This skill relies on the following tools and dependencies:

* Tailwind CSS for utility-first CSS framework
* PostCSS for CSS processing and optimization
* Autoprefixer for prefixing CSS properties
* `clsx` for utility-first class naming
* `tailwind-merge` for merging Tailwind configurations

## Getting Started

To use this skill, follow these steps:

1. **Install Dependencies**: Run `npm install -D @tailwindcss/postcss8 autoprefixer clsx tailwind-merge`
2. **Initialize Tailwind**: Run `npx tailwindcss init -p`
3. **Copy Templates**: Copy `templates/tailwind.config.js` and `templates/globals.css` to your project directory
4. **Import in Your App**: Import `globals.css` in your main entry file or layout component

## Design Tokens System

The design tokens system is based on a set of semantic tokens that define visual properties, including:

* Colors: `--background`, `--surface`, `--text`, `--primary`, `--secondary`, `--accent`
* Typography: `--font-size`, `--line-height`, `--font-weight`
* Spacing: `--spacing-{xs, sm, md, lg, xl, xxl}`

## Component Library

The component library provides a set of reusable components that can be assembled to create complex UI elements. These components include:

* Buttons: `Button`, `LoadingButton`
* Inputs: `Input`, `Textarea`
* Cards: `Card`, `CardGrid`
* Modals: `Modal`, `Alert`

## Accessibility Compliance

This skill ensures that the frontend code meets accessibility standards and guidelines, including:

* Screen reader compatibility
* Keyboard navigation
* High contrast mode
* ARIA attributes

## Testing and Debugging

This skill implements testing strategies and debugging techniques to identify and resolve issues, including:

* Jest for unit testing
* Cypress for end-to-end testing
* Chrome DevTools for debugging

## Configuration

Customize your Tailwind configuration by creating a `tailwind.config.js` file in your project directory.

## Compatibility

This skill is compatible with modern web browsers and supports IE11+.
