# Frontend Design Skill

## Overview

A comprehensive skill for transforming UI style requirements into production-ready frontend code with systematic design tokens, accessibility compliance, and creative execution.

## Skill Location

{project_path}/skills/frontend-design/

## Skill Documentation

### Methodology

This skill follows a structured approach to design and development:

1.  Define the design tokens system
2.  Develop reusable components
3.  Implement utility classes for layout, typography, and states
4.  Integrate the theme management system
5.  Validate against accessibility guidelines

### Design Tokens System

The design tokens system is the foundation of our frontend design. It provides a set of semantic tokens that define the visual properties of our application. These tokens are used throughout the design and development process to ensure consistency and accuracy.

#### Token Types

*   Colors
*   Typography
*   Spacing
*   Radius
*   Shadows
*   Motion

#### Token Values

Each token type has a set of predefined values that can be used throughout the design. These values are defined in the `tokens.css` file.

### Component Development

Reusable components are the building blocks of our frontend design. They provide a way to reuse code and ensure consistency across the application.

#### Component Structure

Each component follows a standard structure:

*   A unique identifier
*   A set of props that define its behavior
*   A set of styles that define its appearance

### Utility Classes

Utility classes are used to add layout, typography, and states to our components. They provide a way to add functionality without duplicating code.

#### Utility Class Types

*   Layout
*   Typography
*   States

#### Utility Class Values

Each utility class type has a set of predefined values that can be used throughout the design. These values are defined in the `utilities.css` file.

### Theme Management

The theme management system provides a way to manage different themes for our application. This allows us to easily switch between different design styles and ensure consistency across the application.

#### Theme Types

*   Light
*   Dark
*   System

#### Theme Values

Each theme type has a set of predefined values that can be used throughout the design. These values are defined in the `theme-provider.tsx` file.

### Accessibility Compliance

Accessibility compliance is a critical aspect of our frontend design. We ensure that our design and development process follows the latest accessibility guidelines to ensure that our application is usable by everyone.

#### Accessibility Guidelines

*   WCAG 2.1
*   Section 508
