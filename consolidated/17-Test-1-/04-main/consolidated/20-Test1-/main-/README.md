## 2. The Sovereign Architecture (Hexagonal Model)

The legacy "Quad-Layer" model has been deprecated in favor of a **Hexagonal Sovereign Architecture**, decoupling core logic from external interfaces to maximize modularity and survival.

### ⬡ The Six Domains

1.  **CORE (The Kernel)**
    *   *Role:* Central processing unit, decision engine, and state manager.
    *   *Path:* `src/core/`
    *   *Logic:* Recursive feedback loops, mutation scheduling.

2.  **CORTEX (Memory & Cognition)**
    *   *Role:* Long-term storage, pattern recognition, strategic planning.
    *   *Path:* `src/cortex/` (connecting to Vector DB/Firebase)
    *   *Feature:* Context-aware retrieval for decision making.

3.  **ARMORY (Tool Registry)**
    *   *Role:* Dynamic capability library.
    *   *Path:* `src/tools/`
    *   *Mechanism:* Hot-swappable modules (Math, API, FileSystem).

4.  **SENTINEL (Governance & Safety)**
    *   *Role:* Immutable constraints and alignment validation.
    *   *Path:* `src/governance/`
    *   *Protocols:* Mission continuity, Regression Guard, Hazard Containment.

5.  **FORGE (Evolution Engine)**
    *   *Role:* Code generation, refactoring, and compilation.
    *   *Path:* `src/evolution/`
    *   *Cycle:* Analyze → Mutate → Test → Deploy.

6.  **NEXUS (Interfaces)**
    *   *Role:* External communication (GitHub, Terminals, APIs).
    *   *Path:* `src/nexus/`
